Порядок работы над задачей
1. Проверка обновлений git pull - сделать на каждой ветке отдельно
2. Проверка состояния git status
3. Создаём новую ветку от мастер 
3a. git checkout master
3b. git checkout -b <новая ветка>
4. Работа в новой ветке
5. Сохранение 
5а. git add .
5b. git commit -a -b "комментарий"
6. Оправка на github: git push --set-upstream origin <имя ветки>
6. если ветку уже отправляли, то: git push
